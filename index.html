My toolchain

(not that you care)

I'm a huge supporter of open source software. Every project of mine is written on an EC2 instance. My development laptop and servers all run archlinux.

I use git for scm, vim for text editing, and tmux for multitasking. You can checkout my dotfiles repo to see what sort of configs I find handy. Vim makes for a great pseudo-IDE.

Any project requiring graphical simulations will likely be written for javascript/canvas.

I prefer node.js and the express framework for web development, though I have been known to revert to python on occasion.

Everything else is written in Haskell. I can't seem to get enough of it.

Current works

I have two projects in progress. A web-based cloud-management platform and an open-source webmail client. These are two products I'm in need of that currently just don't exist in the capacity I'd like them to. If you want to collaborate, by all means get in touch.

winnie

IRC has always been a passion of mine. winnie was a way for me to stay entertained while working. It's an IRC bot that observes a channel and through markov analysis, spews out relevant content later on. It has the eerie allusion of artificial intelligence though it has no real capability for that.

flockr

flockr is my foray into hosting alternative languages in the browser. I re-wrote Craig Renolds' flocking algorithm in various languages to compare their performance. You can check out the ES5 demo here.

GameOfLife

This project was my first attempt at learning coffee-script. I intend to port it to Haskell (via Fay) in the future along with getting a static demo hosted up on github pages at some point.

spider / travelogue

These two projects go hand-in-hand. Spider is a process that simply downloads a webpage or two, indexes them for links, and repeats the process indefinitely. It stores everything in memory (not disk) so it was merely an exercise. Travelogue is a companion project meant to actually index the page text to simulate fast Google-esque searches.
